⑤ 作業フォルダーと初期設定を作る

紫ボックスをクリック → コピーされる → ターミナルに貼ってEnter。

⚡ 作業フォルダーはCドライブに作ろう

OneDriveやクラウド同期フォルダ上で作業すると、ファイルの読み書きが遅くなります。
C:\Users\あなたの名前\Desktop などCドライブ直下に作業フォルダーを置くのがおすすめです。

Claudeへの指示

以下をやってください:
1. Cドライブのデスクトップに作業用フォルダーを作って（フォルダ名は好きに決めてOK。OneDriveの中は避けて）
2. そのフォルダーに移動して /init を実行して
日本語で対応して。

💡 /init って何？ — Claude Codeの初期化コマンドです。今いるフォルダに設定ファイル（CLAUDE.md）を自動で作って、「ここをAIの作業場所にする」と設定してくれます。次のステップでこの指示書を書きます。

Claudeにやってもらう

⑥ 指示書（CLAUDE.md）でAI組織の設計図を書く

CLAUDE.mdに「あなたは何者で、何をすべきか」を書きます。
これがAI組織の設計図になります。

この指示書があるとClaudeが毎回自動で読み込んで、設定なしでAI組織として動きます。

Claudeへの指示（AI組織の設計図を作る）

CLAUDE.md に以下の内容を書いてください:

# AI組織 設定
あなたはCEO（最高執行責任者）として動いてください。
私（人間）は社長です。方針だけ出します。実行は全部あなたがやってください。

## ルール
- 報告は必ず箇条書きで簡潔に
- 専門用語は使わず、わかりやすく説明する
- 外部への送信は私の確認が必要
- タスクが複数あるときはサブエージェントに振って並列処理する
- 毎回「次にやるべきこと」を提案する

## 部署構成
- 秘書: タスク管理・スケジュール・リマインド
- リサーチ: 情報収集・競合調査・ニュース要約
- マーケ: SNS投稿・記事作成・コンテンツ企画
- 開発: コード作成・自動化・ツール構築
- 経理: 請求書・経費・数字の管理

保存したら「設定完了」と教えてください。

起動するたびにAIが設計図を読んで「社長、何をしますか？」という状態になります。

Claudeにやってもらう

⑦ スキル（カスタムコマンド）を作る

繰り返す仕事を /コマンド名 で一発実行できるようにします。
「毎朝やるニュースチェック」「週次レポート」みたいな定型業務をコマンド化して、毎回同じ指示を打つ手間をゼロにします。

Claudeへの指示（スキルを作る）

.claude/commands/ フォルダーに以下のスキルを作ってください:

1.「朝レポート」スキル（ファイル名: 朝レポート.md）:
   実行内容:
   - 最新のAI・ビジネスニュースを3つ取得して要約
   - 今日やるべきタスクを優先度順に整理
   - 「今日の一言アドバイス」を1文で添える
   使い方: /朝レポート と入力するだけで実行

2.「週次レポート」スキル（ファイル名: 週次レポート.md）:
   実行内容:
   - 今週完了したタスクを一覧化
   - 来週の優先タスクをトップ3で提案
   - 改善すべき点を1つ挙げる
   使い方: /週次レポート と入力するだけで実行

作ったら、それぞれの使い方と実行例を教えて。

スキルは一度作ればずっと使えます。定型業務が増えたら同じ方法でどんどん追加できます。

⑧ MCP（外部ツール連携）を設定する

Gmail・Googleカレンダー・Notion・Slackなどと繋げます。
MCPは「コンセントのようなもの」。一度つなげばAIが直接外部ツールを操作できます。

Claudeへの指示（MCP設定）

以下の外部ツール連携（MCP）を設定してください:

1. まずMCPとは何かを30秒で説明して
2. Gmail と連携する手順を教えて（実際に設定まで）
3. Googleカレンダーと連携する手順を教えて
4. 設定が終わったら「カレンダーの今週の予定を教えて」を実行してテストして

日本語でステップバイステップでお願いします。

設定後は「メール確認して」「明日の14時に打ち合わせ入れて」が日本語だけで通じるようになります。

Claudeにやってもらう

⑨ サブエージェント（AI部下）を動かす

Claude Code（CEO）が別のAI（部下）にタスクを投げて同時に処理させる機能です。
人間は大きな方針だけ出す。細かい実行はAIチームが分担します。

Claudeへの指示（サブエージェントで3タスク並列実行）

以下の3つのタスクをサブエージェントに振って、並列で同時に実行してください:

1. リサーチ部: 「AIエージェント」の最新ニュースを3つ調べてまとめて
2. マーケ部: 上のニュースを元にSNS投稿文を3パターン作って
3. 開発部: 「今日のタスクリスト」を管理するHTMLアプリを作って

全部終わったら結果をまとめて一度に報告してください。

3つのタスクが同時並行で進みます。数分で全部の結果が返ってきます。
1人でやると3倍かかる作業が一瞬で終わります。

Claudeにやってもらう

⑩ フック（自動処理）を設定する

特定のアクションが起きたときに自動で動く処理を設定します。
例：タスク完了 → 自動通知、ファイル保存 → 自動バックアップ。
AIが終わったら黙って次の処理まで流してくれる仕組みです。

Claudeへの指示（フック設定）

Claude Codeのフック機能を設定してください:

1. タスクが完了したら自動でサマリーをファイルに保存する
   （保存先: AI作業フォルダー/ログ/完了タスク.txt）

2. エラーが出たら自動でエラー内容をログに記録する
   （保存先: AI作業フォルダー/ログ/エラーログ.txt）

3. ファイルを編集するたびに自動でバックアップを作る
   （保存先: AI作業フォルダー/バックアップ/）

設定方法をステップバイステップで教えて、実際に設定まで全部やってください。

フックを設定すると「あとでやり直せる」安心感が生まれます。AIが大量に動いても追跡できます。

Claudeにやってもらう

⑪ 全部つなげてテスト（AI組織を稼働させる）

これまで設定した全機能（指示書・スキル・MCP・サブエージェント・フック）を組み合わせた総合テストです。
これが通れば、AI組織は完全稼働状態です。

Claudeへの指示（AI組織 本番テスト）

今から本番テストをします。以下を順番に実行してください:

1. /朝レポート を実行して今日のタスクを整理して
2. サブエージェントでリサーチ・マーケ・開発の3タスクを同時実行して
   - リサーチ: 今日のAIニュースを3つ
   - マーケ: そのニュースのSNS投稿文1本
   - 開発: シンプルなメモ帳HTMLファイルを作成
3. 全タスクの結果をAI作業フォルダー/レポート/test_report.txt にまとめて保存して
4. カレンダーに「AI組織テスト完了」として今日の予定を追加して

全部終わったら「テスト完了。AI組織は正常稼働中です」と報告してください。

エラーが出ても大丈夫。Claudeにそのままエラー文を見せれば自動で修正してくれます。

🏢 cc-company — AI仮想会社プラグイン

秘書がヒアリング→CEO が部署に自動振り分け。11部門（PM・マーケ・開発・営業…）が動く仮想会社

以下のコマンドを順番に実行してください:

claude plugin add https://github.com/Shin-sibainu/cc-company

インストールできたら /company を実行して、初期セットアップを開始してください。
秘書が業務内容をヒアリングして、最適な部署構成を自動で作ってくれます。

以下のコマンドを順番に実行してください:

インストールできたら /secretary を実行して、初期セットアップを開始してください。
あなたの役割や業務に合わせて、最適なフォルダ構成とタスク管理を自動で作ってくれます。

設定をパッケージ化（別のPCで5分で再現）

私が使っているスキル・フック・MCP設定をパッケージ化してください:

1. .claude/commands/ の全スキルをリストアップして
2. フック設定を settings.json から読み取ってまとめて
3. MCP設定（.claude/mcp.json）を読み取って
4. 上の3つを「ai_org_package.zip」にまとめて
5. 別のPCでこのパッケージを展開する手順書を README.txt に書いて

これで同じAI組織を別の環境でも5分で再現できるようにする。

スキルを追加する（議事録・日報・提案書）

.claude/commands/ に以下のスキルを追加してください:

2.「日報」スキル（/日報 で実行）:
   今日の作業ログを読んで → 日報フォーマットに自動整理

3.「提案書」スキル（/提案書 で実行）:
   テーマを渡すと → 構成・本文・まとめを自動生成
