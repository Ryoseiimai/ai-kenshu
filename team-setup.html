チームでClaude Codeの設定を共有する方法

プラグイン・スキル・設定をチーム全員で同じにするためのガイド

そもそも「スコープ」って何？

プラグインをインストールするときに「この設定をどこまで適用する？」と聞かれます。

例えるなら、アプリの設定を「自分のスマホだけ」にするか「家族全員のスマホ」にするかを選ぶ感じです。

🙋 Install for you (User Scope) おすすめ

自分のPC全体に適用。どのプロジェクトを開いても使える。個人で使うならこれ一択。

👥 Install for all collaborators (Project Scope)

チームで共有。設定ファイルがプロジェクト内に作られる。チーム全員で同じ設定を使いたいときに選ぶ。

📁 Install for you, in this repo only (Local Scope)

このプロジェクトだけ＆自分だけ。他の人には影響しない。試しに入れたいときに便利。

チームで共有するとどうなる？

Project Scopeを選ぶと、プラグインの設定ファイルがプロジェクトフォルダの中に保存されます。
それをGitHub（ファイルの共有場所）にアップロードすれば、チーム全員が同じ設定を使えるようになります。

あなたのPC

プラグインをインストール（Project Scope）

↓ git push（アップロード）

GitHub（共有場所）

↓ git pull（ダウンロード）

チームメンバーのPC

自動で同じプラグインが使える！

やり方（ステップごと）

あなた（設定する人）がやること

1

プラグインをインストールする

スコープを聞かれたら「Install for all collaborators (Project Scope)」を選ぶ

2

設定ファイルが自動で作られたか確認する

ターミナルで以下を実行：

git status

新しいファイル（.claude/ フォルダなど）が表示されればOK

3

GitHubにアップロードする

以下のコマンドを順番に実行：

git add .

↑ 変更したファイルを全部選択

git commit -m "プラグイン設定を追加"

↑ 「こういう変更をしました」というメモを付けて保存

git push

↑ GitHubにアップロード。これでチームに共有される

チームメンバーがやること

1

最新の設定をダウンロードする

ターミナルで以下を実行：

git pull

↑ GitHubから最新のファイルをダウンロード。プラグインの設定ファイルも一緒に来る

2

Claude Codeを起動する

そのプロジェクトフォルダで claude を実行するだけ。
設定ファイルを自動で読み込んで、同じプラグインが使える状態になります。

つまり、チームメンバーは「git pull → claude」の2コマンドだけ。

よくある質問

Q. チームメンバーもプラグインのインストールが必要？

A. いいえ。Project Scopeで共有された設定は git pull するだけで自動で反映されます。メンバーがインストールコマンドを打つ必要はありません。

Q. User ScopeとProject Scope、両方入れたらどうなる？

A. 両方有効になります。User Scopeは自分だけの設定、Project Scopeはチーム共有の設定として共存できます。

Q. GitHubを使ったことがないんだけど…

A. まずは User Scope（自分だけ）で使ってください。チーム共有はGitHubの基本操作（push/pull）ができるようになってからで大丈夫です。

Q. 後からスコープを変えられる？

A. はい。プラグインをアンインストールして、別のスコープで入れ直せば変更できます。
